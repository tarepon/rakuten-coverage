# モンスターカテゴリ（テーマ）付与 設計書

## 背景・目的

最強プラチナハンターの野生モンスターは、現状すべてのステータス（HP/攻撃/防御）が完全ランダムで、
名前・技も共通プールから抽選されるだけになっている。signalLevel（電波品質）は捕獲率とバトル属性
相性には影響するが、モンスターの「性格」を作っていない。

ポケモンでいう「伝説」「幻」「ロケット団」のように、signalLevelの区分ごとに固有のテーマ性を持たせ、
パラメータ傾向（HP/攻撃/防御のレンジ）と技構成に反映させる。

## 前提の整理（裏取り済み）

楽天モバイルの実際の周波数帯:

| 帯域 | 実体 |
|---|---|
| n77（3.7GHz） | 5G Sub6主力 |
| n257（28GHz） | 5Gミリ波。対応端末が現状ゼロ・エリアも極小のため実質捕獲不可能 |
| Band28（700MHz） | 4Gプラチナバンド（2024年6月開始） |
| n28（700MHz） | 5G版プラチナバンド |

出典:
- [楽天モバイル 取り扱い製品の対応周波数帯一覧](https://network.mobile.rakuten.co.jp/product/frequency-band/)
- [楽天モバイルのプラチナバンドの700MHz帯（バンド28） | Teppei Log](https://teppeilog.com/rakuten_platinum/)

「プラチナ5G」という呼称は公式・各メディアでは確認できず、`SignalLevel.PLATINUM_5G` は実装側の便宜的な命名。
enum名自体は変更せず、テーマ付けのみで対応する。

## 変更① MILLIMETER_WAVE の削除

`SignalLevel.MILLIMETER_WAVE`（n257）は対応端末が存在せずエリアも極小のため、実質到達不可能な区分。
図鑑カテゴリ表示「5Gミリ波モンスター」に該当し、実運用上ほぼ意味を持たないため削除する。

- `SignalLevel` enumから `MILLIMETER_WAVE` を削除
- `resolveSignalLevel()` の `band == "n257"` 分岐を削除
- Room の `signalLevel` カラムは `SignalLevel.name` の文字列保存。万一過去に `"MILLIMETER_WAVE"` 文字列が
  保存されているレコードがあった場合に備え、デシリアライズ側（`CollectionRecord` → `SignalLevel` 変換箇所）に
  未知文字列は `FIVE_G` にフォールバックする処理を1箇所追加する
- `BattleLogic.kt` の `battleType()` / `baseCatchRate()`、`CollectionRecord.kt` の `rarityRank`、
  `CollectionScreen.kt` の表示ラベル・星数・絵文字・色から `MILLIMETER_WAVE` の分岐を削除

## 変更② signalLevelへのテーマ付与

`SignalLevel` の拡張プロパティとして `theme: MonsterTheme` を追加する（1:1対応なので enum 本体は変更せず拡張関数で導出）。

新規 `MonsterTheme` enum:

| enum値 | テーマ名 | 対応するsignalLevel | 実体 |
|---|---|---|---|
| `LEGEND` | 伝説 | `PLATINUM` | LTE Band28（プラチナバンド） |
| `PHANTOM` | 幻 | `PLATINUM_5G` | 5G n28 |
| `ELITE` | 精鋭 | `FIVE_G` | 5G Sub6（n77） |
| `NORMAL` | 通常 | `LTE` | LTE Band3 |
| `ROCKET` | ロケット団 | `WEAK` | auローミング（パートナー回線） |
| `NORMAL_DARK` | 通常（悪） | `NO_SIGNAL` | 圏外 |

`AIRPLANE_MODE` / `NO_SIM` はスタンプ無効（モンスター生成対象外）のため対象外。

## 変更③ パラメータレンジ（テーマ別）

現状 `MonsterGenerator.generate()` は HP/攻撃/防御をそれぞれ独立に `nextInt(100) + 1`（1-100）で決定している。
これをテーマ別のレンジに変更する（レンジ変更方式。乱数消費回数は変えず、`nextInt(bound)` の bound と
オフセットのみ変更する）。

| テーマ | HP | 攻撃 | 防御 | 狙い |
|---|---|---|---|---|
| 伝説（LEGEND） | 70-100 | 70-100 | 70-100 | 全パラメータ最高 |
| 幻（PHANTOM） | 30-60 | 70-100 | 30-60 | 素早さ的位置づけを攻撃に反映、HP/防御低め |
| 精鋭（ELITE） | 50-80 | 70-100 | 40-70 | 攻撃寄り |
| 通常（NORMAL） | 1-100 | 1-100 | 1-100 | 現状維持（バランス型） |
| ロケット団（ROCKET） | 40-70 | 70-100 | 1-40 | 奇襲型・防御低め |
| 通常・悪（NORMAL_DARK） | 1-100 | 1-100 | 1-100 | ステータスは通常種と同格、技のみ悪役寄り |

※本ゲームに「素早さ」ステータスは存在しないため、幻・精鋭の「素早さ特化」は攻撃レンジの底上げで近似する。

## 変更④ 技プール（テーマ別、各10種）

`MonsterAssets.moves` の共通1プール（20種）を廃止し、テーマ別プール（計60種、既存20種＋新規40種）に置き換える。
モンスター生成時はそのモンスターの `theme` に対応するプールからのみ技を抽選する。

技威力は既存の仕組み（技名 → FNVハッシュ → 40-80の範囲に決定論的算出、`movePower()`）をそのまま流用するため、
技名を追加するだけで威力調整は不要。

| テーマ | 技（10種） |
|---|---|
| 伝説 | プラチナアタック、周波数の嵐、スペクトル斬り、シグナルフラッシュ、キングオブバンド、プラチナシールド、ロイヤルウェーブ、グランドカバレッジ、サブロクキング、ソブリンシグナル |
| 幻 | ミリ波ビーム、ビームフォーミング、ゼロレイテンシ、ファントムウェーブ、ミラージュセル、シークレットバンド、ゴーストシグナル、レアバンドフラッシュ、ヴェールオブエヌニハチ、イリュージョンピング |
| 精鋭 | 5Gバースト、キャリアアグリ、ハンドオーバー、サブシックスダッシュ、クイックコネクト、エリートスラッシュ、ハイスピードジャブ、ネットワークソード、ミリ秒キック、アグレッシブビーム |
| 通常 | バンドチェンジ、アンテナ乱舞、電磁波バリア、マクロシールド、スタンダードパンチ、ベーシックウェーブ、デイリーコネクト、ノーマルシグナル、フォースリーパンチ、テッパンリンク |
| ロケット団 | 電波ジャック、ローミングキック、バックホール切断、ジャミングパンチ、パートナーキック、なりすましビーム、カンショウハビーム、ローミングスティール、ネットワークスラム、ブンドリアンテナ |
| 通常（悪） | ピング爆弾、セルタワー落とし、パケット乱射、オニノパケット、ゼロバーズパンチ、ケンガイラッシュ、ダークシグナル、ロストコネクション、サーチングフューリー、ノーバーズキック |

「ミリ波ビーム」はテーマ的に幻（PHANTOM）に割り当てる（元はミリ波区分の技だったが、その区分自体を削除するため）。

## 既存モンスターへの影響

`cellId` は変わらないが、パラメータ生成ロジック・技プール自体が変わるため、既存の捕獲済みモンスターも
（DB上の `hp`/`attack`/`defense`/`moves` を保存している場合はその値のまま、都度生成の場合は次回参照時に）
ステータス・技構成が変わりうる。これは今回の変更の意図した結果として許容する。

## 変更対象ファイル一覧

- `data/Measurement.kt` — `SignalLevel` enumから`MILLIMETER_WAVE`削除、`resolveSignalLevel()`のn257分岐削除
- 新規 `data/monster/MonsterTheme.kt` — `MonsterTheme` enum、`SignalLevel.theme` 拡張プロパティ
- `data/monster/MonsterGenerator.kt` — テーマ別パラメータレンジ・テーマ別技プール抽選ロジック
- `data/monster/MonsterAssets.kt` — `moves` を単一リストからテーマ別マップに再構成
- `data/monster/BattleLogic.kt` — `battleType()` / `baseCatchRate()` から `MILLIMETER_WAVE` 分岐削除
- `data/CollectionRecord.kt` — `rarityRank` から `MILLIMETER_WAVE` 削除、文字列デシリアライズのフォールバック追加
- `ui/collection/CollectionScreen.kt` — 表示ラベル・星数・絵文字・色から `MILLIMETER_WAVE` の分岐削除
