## 2026-07-11 チェックイン刷新・囲って保存・全国データ・ピッカー

### 1. チェックイン機能刷新+「囲って保存」(第1弾)

- **判断**:
  - チェックイン記録は`measurements`混在をやめ専用テーブル`checkin_records`に分離(DB v6)。一覧・スポット別出力がしやすい
  - スピードテストは無料・キー不要の選択肢(Cloudflare/M-Lab NDT7/自前)を比較し、実装が単純で安定なCloudflare公開エンドポイント自前計測を採用(オーナー承認)
  - 写真はCAMERA権限をマニフェスト宣言しない(宣言しなければIntent撮影に権限不要)。TakePicture+PhotoPickerの2系統
  - ラッソはOSMDroidのOverlay継承でタッチ消費、ポリゴン内判定はレイキャスティング法の純粋関数
- **実行**: Sonnetへ詳細仕様を渡して実装→Fableがビルド検証
- **失敗→修正**:
  - `LassoOverlay`の`var enabled`が基底`Overlay.setEnabled(boolean)`とJVMシグネチャ衝突でコンパイルエラー → `active`にリネーム
  - `produceState`がlint(ProduceStateDoesNotAssignValue)に誤検知されやすい書き方 → `remember`+`LaunchedEffect`の定石へ書き換え
  - gradleが「Unable to locate a Java Runtime」→ このMacはAndroid StudioのJBRのみ。`JAVA_HOME=.../Android Studio.app/.../jbr/...`指定+サンドボックス無効で解決(LESSONS.mdにも記録)
- **検証**: assembleDebug/assembleRelease/testDebugUnitTest/lintDebug 全パス

### 2. 実機インストールの断念(→オーナー作業に切り分け)

- **実行**: `adb install -r` でデバッグAPKを実機へ → `INSTALL_FAILED_UPDATE_INCOMPATIBLE`(リリース署名版が既在)
- **判断**: アンインストールは計測データ消失のため**絶対にしない**。署名パスワードはオーナー管理のため、実機確認はオーナーのAndroid Studioビルドで行う運用に確定

### 3. 第2弾(目玉化): ナビ統合・クイック追加・チャート

- **判断**:
  - ナビ6項目はMaterial推奨(≤5)超過 → スタンプはチェックインで獲得するもの=同一ドメインなので「チェックイン画面の2タブ目」に統合し5項目維持(オーナー承認)
  - 「＋」再チェックインの試合フェーズは「当日最新記録の次のフェーズ」を初期値にする自動送り
  - 速度/遅延の色分け閾値: ↓30/5Mbps、↑10/2Mbps、遅延60/150msの3段階
- **失敗→修正**: 特になし(ビルド一発成功)。検証中に既存バグ(API29端末で`CellIdentityNr.bands`クラッシュ)をlintで発見→別セッションに切り出して修正

### 4. 全国スポットデータ

- **判断**:
  - 道の駅は手作業不可(1,145駅) → 国土数値情報P35-18(公式・座標付き・出典表記で同梱可)を採用。座標の創作は絶対禁止をルール化
  - アリーナは2026-27新構造のPREMIER+ONE全クラブ・全ホーム開催会場(52会場)。座標はWikipedia API/OSM実照会からの転記のみ
  - スタンプ記録がspotIdに紐づくため**旧IDを可能な範囲で維持**(道の駅5/5、アリーナ2/5。3会場は本拠地移転で収録外)
- **失敗→修正**:
  - ダウンロードURL確認→P35-18のGeoJSONが同梱されており変換のみで済んだ(GML解析不要)
  - 収録外3会場のスタンプが見えなくなる問題 → スタンプタブを「マスタ全件描画」から「記録ベース表示(進捗バー+達成一覧)」に設計変更して解決(1,145件の性能問題も同時解消)

### 5. ピッカーUI(検索/都道府県/距離/PREMIER・ONE)

- **判断**: 1,145件は全件Compose不可 → LazyColumn必須。都道府県はJIS順。フィルタロジックは純粋関数`SpotFilter`に切り出してテスト可能に
- **実行**: Sonnet実装 → ビルド+テスト43件+lint全パス

### 6. サブエージェント運用の失敗と対策

- **失敗**: Sonnetエージェントが2回「調査をバックグラウンド委譲→自分は停止」し、成果物が出ない状態に(委譲先の結果は親に届かない)
- **修正**: 再開指示に**「委譲・待機を禁止。自分で完遂するまでターンを終了しない」**を明記して解決。以後のデータ収集系の依頼には最初からこの条項を入れる
- **失敗**: セッション上限に2回到達(11時/16:20リセット)して中断 → リセット時刻後に途中成果(スクラッチのOSM照会ダンプ)から再開させ、無駄な再取得を回避

### 7. ネットワーク不調の切り分け(cmdline-tools/システムイメージ)

- **失敗**: dl.google.comからのcurlが32MB/5分の極端な低速で停滞。sdkmanagerのシステムイメージも1.6GBで3時間停滞
- **判断**: Cloudflare/Googleへの単発計測は約600KB/s出る → 回線全体ではなく**TCPセッション単位の固着**と切り分け
- **修正**: 固着したプロセスをkillし、`curl -C -`(部分ファイルからのレジューム)で再取得。sdkmanagerの部分zip(`.temp/PackageOperation01/arm64-v8a-35_r09.zip`)もURL直指定でレジューム

### 8. 実機フィードバック対応

- **指摘**: 検索リストが画面を占領し下部(写真・速度・確定)へ遷移できない
- **判断**: 素の`DropdownMenu`は遅延描画されず1,145件で固まるため不採用。「コンパクトカード+ModalBottomSheet(ほぼ全画面・遅延リスト)」方式に
- **実行**: Sonnet実装→ビルド+テスト全パス→コミット(a10bf44)
- **その他**: リリースビルドの`menuAnchor()`非推奨警告2件 → `MenuAnchorType.PrimaryNotEditable`指定で解消(5bc6898)
